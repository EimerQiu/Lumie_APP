
Smart Ring Bluetooth Protocol					
1	The app sends a 16-byte data packet to the smart ring, and then the smart ring sends back a 16-byte response packet. This communication method is used for all commands except a few specific ones.				
2	The format of the 16-byte data packet sent by the app is as follows:
	B1	B2   ---------------   B15			B16
    B1: Command	
    B2  - B15: Payload bytes, 14 bytes			
    B16: CRC Checksum Byte

	Command: Value range 0x00 to 0x7F; Bit 7 is always 0.	
	The last byte is the CRC. The CRC calculation is simply the sum of all the previous 15 bytes, then taking the lowest 8 bits. For example, if the data packet is: A1 A2 A3 A4... A13 A14 A15 CRC, then CRC = (A1 + A2 + A3 + A4 + ... + A13 + A14 + A15) & 0xFF;				
3	When the smart ring receives a command, it responds with a 16-byte reply data packet. The format is as follows:				
	B1	B2   ---------------   B15			B16
    B1: Command	
    B2  - B15: Payload bytes, 14 bytes			
    B16: CRC Checksum Byte
	Command: Reply with the received command.				
	If the 0x01 command is received and the CRC check is OK, return 0x01. If the CRC is incorrect, set Bit 7 to 1 and return 0x81.				
	The last byte is the CRC. The CRC calculation is simply the sum of all the previous 15 bytes, then taking the lowest 8 bits. For example, if the data packet is: A1 A2 A3 A4... A13 A14 A15 CRC, then CRC = (A1 + A2 + A3 + A4 + ... + A13 + A14 + A15) & 0xFF;				

1	设置时间 Set Time
Command Format: 0x01 AA BB CC DD EE FF 00 00 00 00 00 00 00 00 CRC
Function: Writes the current time from the APP to the ring, 16 bytes (time parameters are entered in decimal)
Description:Received: 5C-00-00-25-08-30-15-03-01-00-63-BE-00-59-00-29-29-2A-0E-38-41-70-90-9B-3D-5C-FF
🏃 Exercise Mode Data (0x5C)
📄 ID: 0000 • 🕒 2025-08-48 21:03:01 • 🏷️ Running • ❤️ 99 bpm
⏱️ 1620:14:54 • 🦶 10496 steps • 🏃 Pace 41:42/km
🔥 2.391934914181951e+29 kcal • 📏 213479253503639552.00 km • ⚠️ CRC mismatch
AA: Year (Year parameter range: 99 ≥ Year ≥ 0. For example, for 2012, AA = 12, default is 21st century, so AA = 12 means the year 2012).
BB: Month (Month parameter range: 1 ≤ Month ≤ 12).
CC: Day (Day parameter should be within the actual number of days in the corresponding month. For example, for January, valid values are from 1 to 31).
DD: Hour (Hour parameter range: 0 ≤ Hour ≤ 23).
EE: Minute (Minute parameter range: 0 ≤ Minute ≤ 59).
FF: Second (Second parameter range: 0 ≤ Second ≤ 59).
Response:
Successful Response (valid checksum and success):
0x01 F4 00 00 00 00 00 00 00 00 00 00 00 00 00 CRC
Failure Response (checksum error or failure in execution):
0x81 00 00 00 00 00 00 00 00 00 00 00 00 00 00 CRC
Example:
Command Sent: 0x01 24 08 08 08 08 08 00 00 00 00 00 00 00 00 CRC
This means the year is 2012, the month is August, the day is 8th, hour is 8, minute is 8, and second is 8.
Response Received: 0x01 F4 00 00 00 00 00 00 00 00 00 00 00 00 00 F5
F4 indicates the maximum Bluetooth data packet length (the maximum data that Bluetooth can send in a single transmission).

2	取得时间 Get time	
Command Format: 0x41 00 00 00 00 00 00 00 00 00 00 00 00 00 00 CRC
Function: The APP retrieves the current time from the ring, a total of 16 bytes (time parameters in the response are in decimal format).
Response:
Successful Response (valid checksum and success):
0x41 YY MM DD hh mm ss ww WW 00 00 00 00 00 00 CRC
Failure Response (checksum error or failure in execution):
0xC1 00 00 00 00 00 00 00 00 00 00 00 00 00 00 CRC
Response Description:
YY: Year (e.g., 12 means 2012, default for the 21st century is 20AA year).
MM: Month (1 to 12).
DD: Day (1 to 31, depending on the month).
hh: Hour (0 to 23).
mm: Minute (0 to 59).
ss: Second (0 to 59).
ww: Weekday (1-7, where 01 is Monday and 07 is Sunday).
WW: Bluetooth data packet maximum length (default is F4, indicating the Bluetooth channel allows sending 244 bytes of data per transmission).
Example:
Command Sent: 0x41 00 00 00 00 00 00 00 00 00 00 00 00 00 00 CRC
The APP requests the current time from the ring.
Response Received: 0x41 24 08 08 08 10 47 06 F4 00 00 00 00 00 00 CRC
24: Year 2024
08: Month 8 (August)
08: Day 8
08: Hour 8
10: Minute 01
47: Second 47
06: Weekday (Saturday)
F4: Bluetooth data packet maximum length (244 bytes)

Response Received: 0x02 00 00 00 00 00 00 00 00 00 00 00 00 00 00 CRC
A successful response indicating the personal information was written to the ring.

11	实时计步/心率/血氧/体温数据传输模式 Real-time Step/HR/Oxygen/Temperature Data Transmission Mode	
Command Format:0x09 AA BB 00 00 00 00 00 00 00 00 00 00 00 00 CRC
Function: Start or stop the real-time step counting mode.
Parameters:
AA:
1: Start real-time step counting.
0: Stop real-time step counting.
BB:
1: Enable real-time temperature.
0: Disable real-time temperature.
Response:
Successful Response (valid checksum and success):
0x09 AA BB CC DD EE FF GG HH II JJ KK LL 00 00 00 00 00 00 00 00 UU T1 T2 S1 00
Failure Response (checksum error or failure in execution):
0xA4 00 00 00 00 00 00 00 00 00 00 00 00 00 00 CRC
Response Description:
AA BB CC DD: Total step count.
EE FF GG HH: Calories (display in KCAL, divide the value by 100 and retain two decimal places).
II JJ KK LL: Walking distance (display in KM, divide the value by 100 and retain two decimal places).
UU: Heart rate value (big-endian format).
T1 T2: Temperature change value (precision of 0.1°C).
S1: Blood oxygen level (requires 0x28 command for data).
Example:
Command Sent: 0x09 01 01 00 00 00 00 00 00 00 00 00 00 00 00 CRC
Response Received: 0x09 66 00 00 00 19 01 00 00 05 00 00 00 25 00 00 00 00 00 00 00 00 5A 01 00 00 00 00 00 00 00 00
66 00 00 00: 102 steps
19 01 00 00: 2.81 KCAL
05 00 00 00: 0.05 km
5A: 90 (Heart rate)
01 00: 1.0°C (Temperature change)
00: Blood oxygen value (requires pairing with 0x28 command for valid data)

12	设置自动检测心率/血氧/HRV等的时间间隔 Set the time interval for automatic detection of heart rate, blood oxygen, HRV, etc.

Function: Set exercise time period.
Description:
AA: Working mode
         0: Off
         2: Interval working mode (automatic measurement interval is set for the entire day, in sleep mode, heart rate interval is automatically set to 5 minutes)
BB: Hour part of the start time for measurements (24-hour format, BCD code format, e.g., 23:00 → AA = 0x23)
CC: Minute part of the start time for measurements, BCD code format (e.g., 59 minutes → CC = 0x59)
DD: Hour part of the end time for measurements (24-hour format, BCD code format, e.g., 23:00 → DD = 0x23)
EE: Minute part of the end time for measurements, BCD code format (e.g., 59 minutes → EE = 0x59)
FF: Weekday enable bits:
Bit0 = 0 means Sunday disabled, Bit0 = 1 means Sunday enabled
Bit1 = 0 means Monday disabled, Bit1 = 1 means Monday enabled
Bit2 = 0 means Tuesday disabled, Bit2 = 1 means Tuesday enabled
Bit3 = 0 means Wednesday disabled, Bit3 = 1 means Wednesday enabled
Bit4 = 0 means Thursday disabled, Bit4 = 1 means Thursday enabled
Bit5 = 0 means Friday disabled, Bit5 = 1 means Friday enabled
Bit6 = 0 means Saturday disabled, Bit6 = 1 means Saturday enabled
GG HH: When working in mode 2, how often to test (in minutes). Each test takes 1 minute.
II: Measurement type
1: Set heart rate
2: Set blood oxygen
4: HRV measurement
Command Response:
If checksum is correct and execution is OK, the response is:
0x2A 00 00 00 00 00 00 00 00 00 00 00 00 00 00 CRC
If checksum is incorrect or execution fails, the response is:
0xAA 00 00 00 00 00 00 00 00 00 00 00 00 00 00 CRC
Example:
Send: 0x2A 02 00 00 23 59 7F 00 05 02 00 00 00 00 00 CRC
Response: 0x2A 00 00 00 00 00 00 00 00 00 00 00 00 00 00 CRC

AA:
99: Delete detailed step count data
00: Read the most recent detailed step count data (retrieve the last 50 data entries based on the timestamp)
YY MM DD HH mm SS: Data timestamp. Sending "00" with a timestamp allows you to query only data after a specified timestamp (for the first sync, return all data, and the timestamp after "00" will be from the historical data).
02: Continue reading from the last read position (retrieve the next 50 entries after the last batch of 50 entries).
Function:
Retrieve detailed step count data.
Response Data Format:
(Variable-length instruction)
0x52 ID1 ID2 YY MM DD HH mm SS S1 S2 K1 K2 D1 D2 SD1 SD2 SD3 ... SD10 (No CRC check)
The minimum length is 25 bytes. A single detailed step count Bluetooth data packet can contain up to 9 data entries. If the packet length is less than the total data being sent, it will be divided into separate packets, with each containing up to 9 entries.
For example, if there are 19 entries, the first packet will contain entries 0-8, the second packet will contain entries 9-17, and the third packet will contain entry 18 followed by a 0x52 FF instruction to indicate the end of the data. The packet division scheme follows a similar rule.
When step counting begins, data is stored every 10 minutes, and the step count for each minute is recorded. If no steps are taken in a minute, it is not counted. (The returned measurement data is in little-endian format.)
Response Data Description:
ID1 ID2: Data read identifier
YY MM DD HH mm SS: Date and time of the data: Year, Month, Day, Hour, Minute, Second.
S1 S2: Step count for the 10-minute period.
K1 K2: Calories burned (in 0.01 kcal).
D1 D2: Distance traveled (in 0.01 km).
SD1 SD2 ... SD10: Step count for each minute in the 10-minute period.
Example:
Send: 0x52 00 00 00 00 00 00 00 00 00 00 00 00 00 00 CRC
Response:

0x52 02 00 24 08 08 23 43 19 0D 00 19 00 01 00 0D 00 00 00 00 00 00 00 00 00

0x52 09 00 24 08 27 09 10 19 12 00 28 00 00 00 0E 04 00 00 00 00 00 00 00 00
0x52 FF
Explanation:
09 00: Data ID.
24 08 27 09 10 19: Date and time is August 27, 2024, at 9:10:19.
12 00: Step count is 33.
28 00: 1.30 kcal burned.
00 00: 0 km traveled.
0E 04 ... 00: Step counts for each minute within the 10-minute period.
Each data packet contains up to 9 data entries.
